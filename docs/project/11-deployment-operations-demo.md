# 11 部署、运维与演示手册

## 1. 部署目标

MVP 采用本地单机部署：

- 一个 Python 虚拟环境；
- 一个 Streamlit 进程；
- 本地数据、模型、缓存和 SQLite；
- 可选外部 OpenAI 兼容 API；
- 浏览器访问 localhost。

这不是生产系统部署手册，不涉及公网、域名、TLS、多用户和高可用。

## 2. 部署包内容

课程演示包应包含：

- 源码；
- 锁定依赖；
- 默认与演示配置；
- 内置 CSV 或明确下载位置；
- 处理后 15 分钟数据；
- 默认模型和缩放器；
- 指标和模型卡；
- 预测、预警和报告缓存；
- SQLite 初始元数据或初始化脚本；
- README 和完整文档；
- 启动脚本；
- 版本清单和文件哈希；
- 不含真实 API Key。

大型模型如不适合直接提交，应提供可靠下载位置、SHA-256 和放置路径。

## 3. 目录与权限

运行账户至少需要：

- 读取源码、配置、内置数据和模型；
- 写入 artifacts、logs 和临时上传目录；
- 访问 localhost 端口；
- 如启用 LLM，访问配置的 HTTPS 地址。

原始 CSV 保持只读。应用启动时检查可写目录，不在首次操作时才暴露权限问题。

## 4. 首次准备

实现后按以下顺序：

1. 按[环境配置](08-technology-and-environment.md)创建虚拟环境；
2. 安装锁定依赖；
3. 复制 .env.example 为 .env，并只在需要时填写 LLM；
4. 校验内置 CSV 指纹；
5. 初始化 SQLite；
6. 生成或核对处理后数据；
7. 放置默认模型、缩放器和模型卡；
8. 生成演示缓存；
9. 运行自动测试；
10. 启动应用并完成验收脚本。

训练是独立步骤，不应在首次启动应用时自动发生。

## 5. 启动

计划命令：

~~~powershell
Set-Location D:\code\AI-Micro-Major-Project
Set-ExecutionPolicy -Scope Process Bypass
.\.venv\Scripts\Activate.ps1
streamlit run app/streamlit_app.py --server.port 8501
~~~

浏览器访问：

~~~text
http://localhost:8501
~~~

实现阶段可以增加 scripts/start_app.ps1，但脚本必须只做检查和启动，不自动安装未知依赖或修改系统设置。

## 6. 启动前检查

- [ ] 电源已连接，Windows 未处于省电模式；
- [ ] 虚拟环境正确；
- [ ] 依赖版本锁定；
- [ ] 内置数据和指纹存在；
- [ ] 处理后数据存在；
- [ ] 默认模型和缩放器存在；
- [ ] 演示缓存存在；
- [ ] artifacts 与 logs 可写；
- [ ] 8501 端口可用；
- [ ] nvidia-smi 正常，或 CPU/缓存方案可用；
- [ ] API 可用，或模板报告已验证；
- [ ] 浏览器缩放和投屏分辨率合适；
- [ ] 关闭游戏、更新程序和占 GPU 软件；
- [ ] 关闭真实密钥可能出现的终端和编辑器窗口。

## 7. 运行模式

### 7.1 开发模式

- 详细日志；
- 页面热重载；
- 可选择小数据；
- 允许显示诊断；
- 不默认缓存所有操作。

### 7.2 演示模式

- 内置数据；
- 预选模型；
- 固定示例日期；
- 优先使用已验证缓存；
- 隐藏调试细节；
- 放大字体；
- 不提供现场训练按钮；
- 大模型按用户点击调用；
- API 失败自动模板。

### 7.3 离线模式

- 禁用所有外部请求；
- 读取本地模型和缓存；
- 使用模板报告；
- 页面显示“离线演示”；
- 仍可完成数据、分析、预测、预警和优化闭环。

## 8. 训练运维

### 8.1 训练前

- 检查数据指纹；
- 记录 Git 提交；
- 检查 GPU 温度、空间和占用；
- 固定配置；
- 运行冒烟训练；
- 确认输出目录全新；
- 记录开始时间。

### 8.2 训练中

- 记录 epoch、训练/验证 loss、MAE、学习率、显存；
- 保存最佳权重；
- 早停；
- OOM 或中断标记 failed；
- 不覆盖以前成功运行。

### 8.3 训练后

- 在固定测试集评估；
- 校准区间；
- 生成模型卡；
- 记录文件哈希；
- 只在门禁通过后注册 candidate；
- 人工选择 default；
- 准备演示缓存；
- 备份完整运行目录。

## 9. 数据更新

导入新数据时：

1. 生成新 dataset_id；
2. 运行完整质量检查；
3. 确认字段和单位；
4. 创建新处理产物；
5. 检查现有模型兼容性；
6. 不兼容则阻止预测；
7. 不自动重训；
8. 若重训，产生新 model_id；
9. 保留旧数据与模型的关联。

不得用新文件覆盖旧文件而继续沿用旧指纹和缓存。

## 10. 备份与恢复

### 10.1 必须备份

- 配置；
- 锁文件；
- 数据 manifest；
- 默认模型、缩放器和模型卡；
- 正式指标；
- 演示缓存；
- SQLite；
- 课程报告、图表和文档；
- 版本清单。

原始 UCI 子集可重新获取，但课程提供文件仍应保留。

### 10.2 备份形式

建议至少两份：

- 本机只读压缩包；
- U 盘或云盘副本。

压缩包命名包含日期和版本，例如 PowerInsight-demo-v1.0-2026xxxx.zip。

### 10.3 恢复演练

在干净临时目录：

1. 解压；
2. 创建环境或使用准备好的离线 wheel；
3. 放置 .env；
4. 校验哈希；
5. 初始化或恢复 SQLite；
6. 启动；
7. 完成用户验收脚本。

答辩前至少做一次恢复演练。

## 11. 日志与诊断

### 11.1 日志位置

计划：logs/app-YYYY-MM-DD.jsonl。

### 11.2 诊断包

可生成脱敏诊断包，包含：

- 应用和依赖版本；
- 操作系统；
- GPU 和 CUDA；
- 数据、模型 ID；
- 非敏感配置；
- 最近错误码；
- 日志尾部；
- 文件存在性。

不包含 Key、Authorization、完整用户数据和完整提示词。

### 11.3 日志保留

默认保留 14 天或最近 20 个运行。课程结果相关日志单独归档。清理不删除被正式报告引用的结果。

## 12. 常见运行故障

### 12.1 应用无法启动

检查虚拟环境、缺少依赖、端口、配置解析和当前目录。先运行最小导入检查，再查看脱敏错误日志。

### 12.2 数据加载慢

确认是否存在处理后 Parquet和缓存；不要每次页面刷新都重新读取并解析完整 CSV。

### 12.3 模型无法加载

检查：

- 文件是否完整；
- 哈希；
- transformers 和 torch 版本；
- model_schema_version；
- 缩放器；
- 输入通道和长度；
- 设备映射。

### 12.4 GPU 不可用

演示立即切换 CPU 或缓存。答辩后再诊断驱动和 wheel，不在现场重装核心依赖。

### 12.5 页面卡顿

缩短显示区间、使用聚合、限制图点、避免重复模型加载、关闭开发热重载。

### 12.6 API 失败

显示失败类别，切换模板报告。不要在现场连续重试消耗时间或额度。

### 12.7 SQLite 锁定

关闭重复应用进程，等待短事务结束。必要时使用备份数据库或只读演示缓存，不强制删除数据库。

## 13. 关闭

1. 确认导出已保存；
2. 停止回放；
3. 在终端按 Ctrl+C 停止 Streamlit；
4. 检查进程退出；
5. 如使用临时上传，按策略清理；
6. 不删除模型、数据库或正式报告；
7. 退出虚拟环境。

## 14. 8 至 10 分钟演示脚本

### 0:00 至 0:45 项目定位

- 说明基于家庭六个月分钟数据；
- 强调完整闭环；
- 介绍三个创新点：PatchTST、共形区间、证据约束大模型。

### 0:45 至 2:00 数据质量

- 加载内置数据；
- 展示 260,640 条；
- 展示 3,771 行缺失和 3,723 分钟长缺失；
- 说明不填 0、不跨长段插值、按时间切分。

### 2:00 至 3:15 用电分析

- 展示总负荷、小时规律、星期热力图；
- 展示分项与未分项电量；
- 点出一条有数据证据的规律。

### 3:15 至 5:15 预测

- 展示模型卡；
- 对比朴素基线、Ridge、LSTM 和 PatchTST；
- 展示未来 24 小时和 90% 区间；
- 强调真实测试集和区间覆盖。

### 5:15 至 6:15 预警

- 回放代表性片段；
- 展示区间外点；
- 打开预警证据；
- 说明统计异常不等于设备故障。

### 6:15 至 7:15 优化

- 输入峰谷价格和转移比例；
- 运行情景；
- 展示峰值、费用、转移量和能量守恒；
- 说明只模拟不控制。

### 7:15 至 8:30 大模型报告

- 预览发送证据；
- 生成报告；
- 展示 evidence_id；
- 切换或演示模板回退。

### 8:30 至 9:30 总结

- 回顾功能闭环；
- 说明本地 4060 可训练；
- 说明数据和模型限制；
- 展示导出结果。

保留 30 秒用于缓冲或教师提问。

## 15. 演示故障兜底表

| 故障 | 立即动作 | 可继续展示 |
| --- | --- | --- |
| 无网络 | 切离线模式 | 全部核心功能，报告用模板 |
| API 错误 | 展示错误类别并回退 | 是 |
| GPU 不可用 | 加载缓存或 CPU | 是 |
| 即时预测慢 | 加载已验证 forecast_id | 是 |
| 数据处理慢 | 使用处理后 Parquet | 是 |
| 模型文件损坏 | 使用备份模型或缓存 | 大部分 |
| 端口冲突 | 改为 8502 | 是 |
| 浏览器异常 | 换浏览器或重开 localhost | 是 |
| 数据库损坏 | 使用备份或只读缓存 | 大部分 |
| 投屏分辨率异常 | 展示模式、浏览器缩放 | 是 |
