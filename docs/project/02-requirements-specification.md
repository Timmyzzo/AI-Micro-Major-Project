# 02 需求规格说明书

## 1. 目的与适用范围

本文档把项目目标转化为可开发、可测试的需求。需求编号在界面、测试、提交记录和课程报告中保持稳定。

优先级定义：

- P0：课程 MVP 必须完成。
- P1：应完成；可在不影响 P0 时实现。
- P2：扩展项；不作为 MVP 验收前提。

## 2. 系统角色

MVP 只有一个本地“操作用户”角色，不做登录和权限。操作用户可导入数据、加载模型、运行分析、配置 API、查看预警和导出结果。

大模型服务、文件系统和 GPU 是外部依赖，不是系统用户。

## 3. 功能需求

### 3.1 数据接入与数据治理

#### FR-DATA-001 使用内置数据集

- 优先级：P0
- 前置条件：内置 CSV 存在且校验通过。
- 行为：用户选择“内置演示数据”，系统读取文件元数据、字段、时间范围和数据指纹。
- 输出：数据集名称、记录数、时间范围、采样频率、缺失率和载入状态。
- 验收：
  - 能识别 260,640 条记录。
  - 能显示 2007-01-01 至 2007-06-30。
  - 能显示 SHA-256 指纹的缩略值和完整复制入口。
  - 原文件不被修改。

#### FR-DATA-002 上传 CSV

- 优先级：P0
- 输入：一个 CSV 文件。
- 行为：先读表头和样本，再进行完整校验；上传失败不影响内置数据。
- 约束：默认文件上限建议 200 MB，可通过配置调整。
- 验收：
  - 支持 UTF-8 和常见英文逗号分隔格式。
  - 字段缺失时给出缺少字段列表和修复建议。
  - 不把无效文件静默当作空数据。

#### FR-DATA-003 字段映射

- 优先级：P1
- 行为：对与标准名称不一致的上传文件，允许用户把时间、总有功功率和可选辅助字段映射到系统字段。
- 最低要求：时间戳和目标列必须可确定。
- 验收：保存映射方案并在导入摘要中展示。

#### FR-DATA-004 数据质量检查

- 优先级：P0
- 检查项：表头、类型、时间解析、时间排序、重复、采样间隔、缺失、无穷值、明显越界和常量列。
- 输出：总评分、错误、警告、信息三级问题，以及问题时间范围。
- 验收：内置数据应识别 3,771 行问号缺失和最长 3,723 分钟缺失段。

#### FR-DATA-005 预处理预览

- 优先级：P0
- 行为：展示原始值、清洗后值、缺失掩码和 15 分钟聚合样例。
- 验收：用户可以看到每项规则处理了多少行；长缺失段不得被静默跨段插值。

#### FR-DATA-006 数据集版本

- 优先级：P0
- 行为：为每个导入数据生成 dataset_id、SHA-256、导入时间和配置摘要。
- 验收：模型运行、预测和报告均引用 dataset_id。

### 3.2 用电监测与探索分析

#### FR-MON-001 总览指标

- 优先级：P0
- 指标：数据时间范围、最近有效负荷、所选区间平均负荷、峰值、总有功电量、缺失率和异常数。
- 验收：所有指标有单位、统计区间和说明提示。

#### FR-MON-002 历史趋势

- 优先级：P0
- 行为：按日期范围和粒度显示总有功功率、无功功率、电压、电流及分项计量。
- 交互：缩放、悬停、图例开关、恢复视图。
- 验收：大范围查询使用聚合数据，页面不一次绘制 26 万个点导致卡顿。

#### FR-MON-003 周期规律

- 优先级：P0
- 图表：小时平均曲线、星期分布、日历热力图和工作日/周末对比。
- 验收：图表标明聚合方法和缺失处理。

#### FR-MON-004 分项用电结构

- 优先级：P0
- 行为：展示厨房、洗衣房、热水器/空调和未分项电量构成。
- 验收：使用正确单位；未分项电量按数据文档公式计算。

#### FR-MON-005 相关性与典型日

- 优先级：P1
- 行为：展示数值特征相关矩阵，并选择高负荷日、低负荷日和典型日进行对比。
- 限制：相关性不得解释为因果关系。

#### FR-MON-006 模拟实时回放

- 优先级：P0
- 输入：起始日期、回放速度和步长。
- 行为：从历史数据逐点或逐批推进当前时间，刷新指标、曲线和预警。
- 验收：支持暂停、继续、重置；回放不改变原始数据。

#### M3 实现状态（2026-07-21）

- FR-MON-001：已实现所选范围累计有功电量、平均/峰值/最低有效功率、峰谷时间、理论/实际/有效/缺失点数和覆盖率；未知值显示为“—”。
- FR-MON-002：已实现基于 15 分钟 Parquet 的总有功功率历史趋势、日期范围与确定性限点；无功功率、电压、电流和可选分项趋势仍未纳入本阶段。
- FR-MON-003：已实现小时、星期一至星期日、工作日/周末日内对比，并显示每桶有效样本数和覆盖率；日历热力图仍为后续分析扩展。
- FR-MON-004：已实现三个分项和未分项累计能量；M2 记录的 11 条负未分项分钟原值继续作为质量证据保留，因此当前不生成占比。
- FR-MON-005 与 FR-MON-006：相关性、典型日和模拟实时回放仍未实现，不属于本次独立 M3 最小闭环。

M3 所有数值均来自当前 `power_15min.parquet`，没有读取原始 CSV 重新聚合、训练模型、生成未来预测或调用外部 API。

### 3.3 负荷预测

#### FR-FCST-001 选择预测时间

- 优先级：P0
- 输入：具有至少 7 天有效历史的预测起点。
- 行为：构造 672 个 15 分钟上下文点，输出未来 96 点。
- 验收：历史不足或跨越长缺失段时阻止预测并说明原因。

#### FR-FCST-002 加载模型

- 优先级：P0
- 行为：显示模型名称、版本、数据版本、训练区间、指标、设备和加载状态。
- 验收：模型与当前数据字段或缩放器不兼容时拒绝加载。

#### FR-FCST-003 基线对比

- 优先级：P0
- 基线：P0 包含前一时刻、前一日同刻、前一周同刻和 Ridge；LSTM 作为 P1 的课程实验关联基线。
- 验收：同一测试区间、同一指标、同一数据处理下比较。

#### FR-FCST-004 多步预测

- 优先级：P0
- 输出：时间戳、点预测、下界、上界、模型与 run_id。
- 验收：完整输出 96 个连续 15 分钟点；图表区分历史、真实、预测和区间。

#### FR-FCST-005 预测指标

- 优先级：P0
- 指标：MAE、RMSE、WAPE、sMAPE、R²，以及按预测步长误差。
- 验收：指标只在测试集或明确的回测区间计算，不使用训练集冒充效果。

#### FR-FCST-006 预测缓存与导出

- 优先级：P0
- 行为：保存预测结果和元数据，支持 CSV 导出。
- 验收：离线演示能加载事先缓存结果，并明确标记“缓存”。

### 3.4 异常监测与预警

#### FR-ALERT-001 数据质量预警

- 优先级：P0
- 触发：缺失、重复、时间断点、无效值和字段异常。
- 输出：类型、开始、结束、影响行数、等级和处置建议。

#### FR-ALERT-002 规则预警

- 优先级：P0
- 触发：负荷、变化率、电压或电流超过用户阈值或训练数据稳健阈值。
- 限制：默认阈值是数据异常阈值，不是电气安全法规阈值。

#### FR-ALERT-003 预测残差预警

- 优先级：P0
- 触发：观测值落在相应预测区间之外，或标准化残差超过等级阈值。
- 输出：观测、预测、区间、残差、异常分数和模型运行 ID。

#### FR-ALERT-004 预警分级

- 优先级：P0
- 等级：信息、关注、严重。
- 规则：等级由确定性规则计算，不由大模型自由判断。
- 验收：相同数据和配置产生相同等级。

#### FR-ALERT-005 预警管理

- 优先级：P1
- 行为：按时间、类型、等级和状态筛选；标记未处理、已确认、已忽略；添加本地备注。
- 限制：状态仅用于演示，不代表真实运维闭环。

#### FR-ALERT-006 预警导出

- 优先级：P0
- 格式：CSV；报告中可生成摘要表。
- 验收：导出包含阈值、证据和配置版本。

### 3.5 优化与决策支持

#### FR-OPT-001 峰谷参数

- 优先级：P0
- 输入：峰、平、谷时间段及单价。
- 行为：校验时间段覆盖与价格非负。
- 限制：默认参数仅用于演示，不宣称为数据采集地真实电价。

#### FR-OPT-002 可转移负荷情景

- 优先级：P0
- 输入：目标日期、可转移分项、可转移比例、允许时间窗和最大延迟。
- 行为：把指定能量从高负荷时段移动到允许的低负荷时段，保持总能量守恒。
- 验收：
  - 调整前后总能量差在数值容差内。
  - 显示峰值、峰谷差和模拟费用变化。
  - 明确标记“情景模拟，未执行设备控制”。

#### FR-OPT-003 方案比较

- 优先级：P1
- 行为：保存并比较最多三个方案，展示雷达图或表格。
- 指标：峰值、峰谷差、费用、转移能量、受影响时段。

#### FR-OPT-004 建议生成

- 优先级：P0
- 行为：规则引擎先形成事实和约束，大模型可将其改写为自然语言建议。
- 验收：关闭大模型后仍有模板建议。

### 3.6 大模型智能分析

#### FR-LLM-001 API 配置

- 优先级：P0
- 配置：启用开关、base URL、API Key、模型名、接口风格 auto/chat_completions/responses、超时、温度。
- 安全：Key 只从环境变量或本地会话读取，默认不回显、不入日志、不入导出。

#### FR-LLM-002 连接测试

- 优先级：P0
- 行为：用最小请求测试兼容性和延迟。
- 输出：成功、鉴权失败、模型不存在、超时、限流或格式不兼容。
- 验收：失败信息可理解且不显示 Key。

#### FR-LLM-003 证据上下文

- 优先级：P0
- 输入：数据质量、核心 KPI、预测摘要、异常列表、优化情景和限制。
- 限制：默认不发送整份 CSV，不发送超过配置数量的逐点数据。
- 验收：调用前可预览将发送的脱敏 JSON。

#### FR-LLM-004 结构化报告

- 优先级：P0
- 输出章节：执行摘要、数据质量、负荷规律、预测与风险、异常解释、优化建议、限制与免责声明。
- 验收：每项结论引用 evidence_id；缺乏证据时输出“无法判断”。

#### FR-LLM-005 失败回退

- 优先级：P0
- 触发：未配置、超时、限流、服务错误、JSON 不合法或内容校验失败。
- 行为：生成确定性模板报告，并保留错误类别供诊断。
- 验收：核心页面不崩溃，用户仍能导出报告。

#### FR-LLM-006 报告导出

- 优先级：P0
- 格式：Markdown；可选 HTML。
- 内容：生成时间、数据版本、模型版本、是否使用大模型、提供商配置别名和免责声明。

### 3.7 系统设置与可追溯

#### FR-SYS-001 配置管理

- 优先级：P0
- 配置域：数据、预测、预警、优化、大模型、界面。
- 行为：显示默认值、当前值和来源；危险或无效值不能保存。

#### FR-SYS-002 运行诊断

- 优先级：P0
- 内容：Python、PyTorch、CUDA 是否可用、GPU 名称、内存、模型文件、数据文件、数据库和 API 状态。
- 安全：不显示完整 Key、用户目录敏感信息或请求正文。

#### FR-SYS-003 运行追踪

- 优先级：P0
- 行为：为预处理、训练、评估、预测和报告生成唯一 ID，保存关键配置和时间。
- 验收：从报告可追溯到预测和数据版本。

#### FR-SYS-004 展示模式

- 优先级：P1
- 行为：隐藏调试细节、放大字体、使用固定演示数据和缓存结果。
- 验收：可一键恢复普通模式。

## 4. 非功能需求

### 4.1 性能

| 编号 | 要求 |
| --- | --- |
| NFR-PERF-001 | 内置数据初次载入和质量检查目标不超过 15 秒，后续缓存载入不超过 5 秒；以目标设备实测为准 |
| NFR-PERF-002 | 常用图表交互目标响应不超过 2 秒 |
| NFR-PERF-003 | 单次 96 点 GPU 推理目标不超过 2 秒，CPU 推理目标不超过 10 秒 |
| NFR-PERF-004 | 大模型报告默认超时 30 秒，可配置；超时后回退 |
| NFR-PERF-005 | 绘图前进行聚合或下采样，单图建议不超过 10,000 点 |

### 4.2 资源

| 编号 | 要求 |
| --- | --- |
| NFR-RES-001 | 默认训练配置适配 8 GB 级显存 |
| NFR-RES-002 | 支持自动混合精度和梯度裁剪 |
| NFR-RES-003 | CUDA 不可用时允许 CPU 推理和最小训练冒烟测试 |
| NFR-RES-004 | OOM 时给出减小 batch size、模型维度或上下文的建议 |

### 4.3 可用性

- NFR-USE-001：界面和报告使用中文，技术缩写首次出现时给出中文解释。
- NFR-USE-002：所有指标显示单位、时间范围和计算口径。
- NFR-USE-003：空状态、加载状态、成功、警告和错误状态均有明确反馈。
- NFR-USE-004：关键操作不依赖命令行。
- NFR-USE-005：1920×1080、100% 至 150% 系统缩放下可正常展示。
- NFR-USE-006：颜色不是表达异常等级的唯一方式，同时使用图标和文字。

### 4.4 可靠性与复现

- NFR-REL-001：固定随机种子并记录库版本、数据指纹和配置。
- NFR-REL-002：大模型失败不得影响确定性分析结果。
- NFR-REL-003：原始数据只读，处理产物写入独立目录。
- NFR-REL-004：缓存有 schema_version，版本不兼容时自动失效。
- NFR-REL-005：训练异常中断后不把半成品标记为最佳模型。

### 4.5 安全与隐私

- NFR-SEC-001：API Key 不写入源码、SQLite、日志、截图和导出。
- NFR-SEC-002：大模型默认只接收聚合摘要和最多若干异常样本。
- NFR-SEC-003：上传文件仅在本机处理，除非用户主动调用外部大模型。
- NFR-SEC-004：对 CSV 公式注入字符进行导出防护。
- NFR-SEC-005：系统报告包含“教学演示，不作为安全或调度依据”声明。

### 4.6 可维护性

- NFR-MNT-001：业务逻辑不直接写在页面回调中，按数据、模型、预警、决策和 LLM 模块分离。
- NFR-MNT-002：公共函数有类型标注和文档字符串。
- NFR-MNT-003：Ruff、mypy 和 pytest 作为质量门禁。
- NFR-MNT-004：配置集中管理，不散落魔法数字。
- NFR-MNT-005：需求变更同步更新文档和测试。

## 5. 业务规则

| 编号 | 规则 |
| --- | --- |
| BR-001 | 日期按日/月/年解释，不能使用默认月/日推断 |
| BR-002 | 问号和无法解析的数值视为缺失，绝不直接填 0 |
| BR-003 | 长缺失段不进行跨段插值，相关训练窗口剔除或显式掩码 |
| BR-004 | 标准化器只在训练集拟合 |
| BR-005 | 模型选择只使用验证集，测试集只用于最终一次评估 |
| BR-006 | 预测区间由验证残差校准，不能由大模型编造 |
| BR-007 | 预警等级由规则和数值分数计算 |
| BR-008 | 优化情景前后总能量应守恒，除非明确启用节能比例假设 |
| BR-009 | 大模型报告必须携带证据 ID 和限制 |
| BR-010 | 缓存结果必须标明生成时间、数据和模型版本 |

## 6. 数据与结果保留

- 原始课程 CSV 永久保留且只读。
- 上传临时文件默认在会话结束后可清理。
- 处理后数据、模型、评估结果和演示缓存按 run_id 分目录保存。
- 日志默认保留 14 天或最近 20 个运行；课程提交前可归档。
- 大模型请求默认不保存完整提示词中的敏感字段，只保存模板版本、响应状态和报告文件。

## 7. 需求依赖

| 需求 | 依赖 |
| --- | --- |
| 预测 | 已完成数据校验、预处理、模型训练或存在兼容缓存 |
| 残差预警 | 同一时间点有预测和观测 |
| 优化模拟 | 有负荷曲线、可转移分项或用户指定的可转移比例 |
| 大模型报告 | 有确定性分析结果；API 仅影响生成方式 |
| 展示模式 | 有可用的演示数据、模型和缓存 |
