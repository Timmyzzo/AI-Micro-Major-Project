# 02 需求规格

## P0 功能需求

### DATA 当前案例数据

- FR-DATA-001：当前可复现版本使用 `docs/household_power_consumption.csv`；课程方法不限制数据只能来自该文件。
- FR-DATA-002：校验字段、时间、重复、缺失、采样间隔和明显无效值。
- FR-DATA-003：保留原始文件只读，生成分钟和 15 分钟 Parquet、manifest 与稳定 ID。
- FR-DATA-004：短缺失可插值，长缺失不得跨段填充。
- FR-DATA-005：若开发阶段更换为公开电力数据，必须明确来源、许可、字段、单位、时间粒度和按时间切分规则，并重新生成数据身份、处理产物、模型和测试结果。

当前 UI 不提供上传、字段映射、多数据集或数据版本切换；这不代表课程只能使用当前 CSV。

### MON 历史分析

- FR-MON-001：展示区间覆盖率、累计电量、平均、峰值和最低功率。
- FR-MON-002：展示确定性历史趋势和周期规律。
- FR-MON-003：展示三个分项和未分项能量证据。

### FCST 负荷预测

- FR-FCST-001：使用 672 个 15 分钟历史点预测未来 96 点。
- FR-FCST-002：比较朴素基线、Ridge、LSTM 和 PatchTST。
- FR-FCST-003：训练集用于拟合权重和缩放器；验证集用于神经网络早停、Ridge alpha 等内部选参；所有候选模型冻结后，测试集用于统一比较和结果展示。
- FR-FCST-004：输出点预测、90% 区间、真实值和区间外标记。
- FR-FCST-005：支持冻结模型加载、离线缓存和 CSV 导出。
- FR-FCST-006：保存 LSTM/PatchTST 的 epoch、训练损失、验证 MAE、训练耗时和配置快照，并在课程文档中展示训练过程。
- FR-FCST-007：回归效果使用 MAE、RMSE、WAPE、sMAPE、R²和区间覆盖率；不得把未定义的 `1 - error` 直接称为 Accuracy。需要百分比命中率时，必须给出明确的绝对误差阈值和计算公式。

当前 M4 注册表将测试 MAE 最低候选标记为页面“展示默认”。该标记用于演示导航，不得写成“测试集完全未参与模型选择”；若后续需要严格无偏的部署模型选择，应在查看测试结果前按验证指标确定默认模型，或另设最终留出集。

### ALERT 回放与预警（扩展）

- FR-ALERT-001：对一个 96 点历史回测片段继续、暂停、单步和重置。
- FR-ALERT-002：把质量问题转换为质量预警。
- FR-ALERT-003：使用训练段稳健阈值生成负荷和变化规则预警。
- FR-ALERT-004：真实值落在预测区间外时生成残差预警。
- FR-ALERT-005：等级由确定性规则计算并支持 CSV 导出。

不实现确认、忽略、备注或预警数据库。

### ADVICE 外部大模型建议

- FR-ADVICE-001：首页显示大模型启用状态、模型型号和一键连接测试。
- FR-ADVICE-002：用户主动点击后调用一次 Chat Completions，生成内容较完整的中文建议。
- FR-ADVICE-003：发送内容只包含数据身份、时间范围、覆盖率、电量、平均/峰值负荷、默认模型及测试 MAE和限制。
- FR-ADVICE-004：不得发送原始 CSV、完整时序、Key、权重或本地路径。
- FR-ADVICE-005：API 未配置、失败或返回空内容时显示明确状态，不伪装成调用成功。
- FR-ADVICE-006：建议按现状判断、峰值管理、日常执行和持续观察组织，并支持一键导出 Markdown。

不实现完整报告、Responses、自动探测、结构化输出、HTML、历史记录或多轮对话。

## 非功能需求

- 页面刷新不训练模型、不自动调用 API。
- 原始数据只读，Key 不写入 YAML、日志、SQLite、导出或截图。
- 相同数据和配置产生相同分析、预测缓存与预警 ID。
- 颜色不是状态的唯一表达方式。
- Ruff、格式、mypy、pytest 和 pre-commit 必须通过。
- 页面语言面向普通用户和答辩展示，不出现开发阶段编号、内部产物名称或方法论免责声明堆叠。

数据采集、连续实时监测和优化调度不属于本次课程核心验收要求。
