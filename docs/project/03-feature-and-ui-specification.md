# 03 页面与交互

## 七页信息架构

1. 首页总览：数据、分析、预测、预警和大模型 API 状态；显示模型型号并提供一键连接测试。
2. 数据中心：UCI 数据来源、时间范围、数据规模、覆盖率和数据示例；缺失时仅提供“准备分析数据”。
3. 用电分析：日期选择、核心指标、趋势、周期和分项结构。
4. 负荷预测：预测时间、三个主要模型、24 小时预测、双语指标和 CSV 下载。
5. 监测预警：历史回放、异常数量、类型、等级、记录和 CSV 下载。
6. 智能建议：四项用电摘要、当前模型、一个生成按钮和大模型回复。
7. 系统设置：Python、PyTorch、GPU、数据、三个预测模型、大模型状态和型号。

## 展示语言

- 页面和侧栏不出现 M1、M2、M3、M4、M5；
- 不向用户展示哈希、dataset_id、preprocess_id、run_id、manifest、配置指纹或代码提交；
- 不展示“页面不训练”“不调用 LLM”“安全边界”“仅供课程演示”等研发说明；
- 每页只突出一个主任务，标题说明不超过一句；
- 操作必须显示进行中、成功或失败状态。

## 模型展示

负荷预测页只展示：昨日同刻基线、LSTM 神经网络和 PatchTST 深度模型。训练脚本可以保留其他候选用于内部比较，但不占用展示界面。

应用内训练曲线仍未实现；逐轮训练损失和验证误差保存在模型指标并写入课程报告。

## 大模型交互

- 外部大模型 API 是正式功能；
- 默认启用 LLM 意图，但 Key 或模型暂缺时系统仍可打开；
- 首页始终显示 API 状态和“测试 API 连接”按钮；
- 智能建议页始终显示“生成智能建议”按钮；
- 成功显示模型回复，失败显示简短错误；
- 页面加载本身不自动发起网络请求。
