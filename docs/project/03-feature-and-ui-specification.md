# 03 功能与界面规格

## 1. 设计目标

界面应让非算法专业的教师在 30 秒内理解“数据是否可靠、负荷有什么规律、未来会怎样、哪里异常、可以怎么优化”。系统采用 Streamlit 多页面单体和 Plotly 交互图表，重点是信息层级、证据链和演示稳定性，而不是堆叠功能。

## 2. 信息架构

~~~mermaid
flowchart LR
    A["首页总览"] --> B["数据中心"]
    B --> C["用电分析"]
    C --> D["负荷预测"]
    D --> E["监测预警"]
    E --> F["优化决策"]
    F --> G["智能报告"]
    G --> H["系统设置"]
~~~

侧边栏用于全局数据集、时间范围、模型和展示模式；页面内部只放与当前任务相关的筛选条件。

## 3. 全局布局

### 3.1 顶部区域

- 左侧：项目名称和当前页面。
- 中部：当前数据集、时间范围、数据状态。
- 右侧：GPU/API 状态、展示模式开关、帮助入口。
- 状态标签应使用“就绪、处理中、需关注、失败、缓存”明确描述，不能只显示颜色圆点。

### 3.2 侧边栏

固定顺序：

1. 数据源选择；
2. 时间范围；
3. 数据粒度；
4. 当前模型；
5. 快捷操作；
6. 系统状态。

侧边栏不直接放 API Key。密钥配置只在系统设置的受控输入框中完成。

### 3.3 页面主体

推荐使用“标题与说明 → 筛选器 → KPI → 主图 → 证据表 → 方法与限制”的统一结构。每张图必须有：

- 标题；
- 时间范围；
- 单位；
- 聚合口径；
- 悬停说明；
- 数据缺失提示；
- 下载入口或可导出数据的说明。

## 4. 视觉规范

### 4.1 主题

- 默认：深色科技风，适合投屏。
- 可选：浅色报告风，适合截图和打印。
- 背景避免纯黑，使用深蓝灰；卡片之间留出稳定间距。

### 4.2 建议色板

| 含义 | 颜色 | 用法 |
| --- | --- | --- |
| 主色 | #3B82F6 | 按钮、选中状态、主预测线 |
| 辅色 | #14B8A6 | 正常、节能、对比线 |
| 历史观测 | #94A3B8 | 历史负荷 |
| 预测区间 | rgba(59,130,246,0.18) | 90% 区间填充 |
| 信息 | #38BDF8 | 数据质量提示 |
| 关注 | #F59E0B | 中等级预警 |
| 严重 | #EF4444 | 严重预警 |
| 禁用/未知 | #64748B | 无数据、禁用功能 |

同一语义在所有页面使用同一颜色。异常还要配合图标、文字和线型，照顾色觉差异。

### 4.3 字体与数字

- 中文字体优先使用系统可用的微软雅黑或思源黑体，回退到无衬线字体。
- 页面标题 28 至 32 px，分区标题 20 至 24 px，正文不低于 14 px。
- KPI 数字保留合理小数：功率 2 位、电量 1 位、百分比 1 位、费用 2 位。
- 千位使用分隔符；日期统一为 YYYY-MM-DD HH:mm。

## 6. 页面规格

### 6.1 首页总览

#### 目标

在一屏内表达系统当前状态和最重要结论。

#### 顶部 KPI

1. 数据完整度；
2. 当前或最近负荷；
3. 所选区间峰值及发生时间；
4. 所选区间总有功电量；
5. 活跃预警数量；
6. 未来 24 小时预测峰值。

每个 KPI 卡片显示单位、与上一可比区间的变化、数据时间和信息提示。

#### 主内容

- 左侧宽图：历史 24 小时与未来 24 小时负荷，包含预测区间。
- 右上：预警等级分布和最新三条预警。
- 右下：分项用电构成。
- 底部：数据质量摘要、模型卡摘要和“一键生成报告”入口。

#### 状态

- 未选数据：显示引导卡。
- 数据有阻断错误：预测卡禁用，突出数据中心入口。
- 无模型：显示“尚无可用模型”，允许展示历史分析。
- 使用缓存：图表右上明确标记缓存生成时间。

### 6.2 数据中心

#### 数据源区

- 内置数据卡：名称、大小、记录数、校验状态、加载按钮。
- 上传区：拖放或选择文件，显示大小限制和必要字段。
- 最近数据集：dataset_id、导入时间、来源和状态。

#### 字段与质量区

- 字段表：原列名、标准字段、类型、单位、非空数、缺失率、最小、最大、均值。
- 时间检查：起止时间、理论频率、重复、断点、乱序。
- 缺失区段：开始、结束、长度、是否可插值。
- 质量卡：完整性、有效性、唯一性、时间完整性、长缺失警告和总状态。

#### 预处理对比

- 原始与清洗后的同区间双线图。
- 处理步骤时间线：解析 → 缺失识别 → 短缺失插值 → 聚合 → 特征生成。
- 每步显示输入行数、输出行数、变化数和配置。

#### 操作

- 运行校验；
- 生成预处理数据；
- 下载质量报告；
- 复制数据指纹；
- 重置为内置数据。

### 6.3 用电分析

#### 筛选器

- 日期范围；
- 粒度：15 分钟、小时、日；
- 指标；
- 工作日/周末；
- 是否显示缺失阴影。

#### 图表

1. 总负荷趋势；
2. 小时轮廓曲线及分位带；
3. 星期×小时热力图；
4. 日历热力图；
5. 分项用电堆叠面积图；
6. 相关性矩阵；
7. 高、低、典型日对比。

#### 分析摘要

由确定性规则生成，例如“本区间峰值出现在 20:15”“周末平均负荷比工作日高 X%”。这些摘要是大模型输入证据，不依赖大模型。

### 6.4 负荷预测

#### 控制区

- 预测起点；
- 模型选择；
- 设备选择：自动、CUDA、CPU；
- 区间置信水平，MVP 默认 90%；
- 运行预测或加载缓存。

#### 模型卡

- 模型类型和版本；
- run_id；
- 训练、验证、测试月份；
- 上下文与预测长度；
- 数据指纹；
- 测试指标；
- 是否为当前默认模型；
- 已知限制。

#### 主图

- 历史上下文：灰色实线；
- 真实未来：仅回测时显示，深色实线；
- 点预测：蓝色实线；
- 区间：浅蓝填充；
- 区间外观测：红色点；
- 长缺失区域：灰色阴影。

#### 评估区

- 模型对比表；
- MAE、RMSE、WAPE、sMAPE、R²；
- 1、6、12、24 小时预测误差；
- 按步长误差曲线；
- 区间覆盖率与平均宽度。

不得只显示训练损失。测试区间必须明确。

### 6.5 监测预警

#### 回放控制

- 起始日期；
- 速度：1×、10×、60×、逐小时；
- 播放、暂停、单步、重置；
- 当前模拟时间。

#### 实时区域

- 当前负荷、电压、电流；
- 预测中位数与区间；
- 标准化残差；
- 最近 6 小时曲线；
- 当前等级和触发规则。

#### 预警列表

| 字段 | 展示 |
| --- | --- |
| 时间 | 开始与结束 |
| 等级 | 信息、关注、严重 |
| 类型 | 数据质量、规则、预测残差 |
| 证据 | 观测、阈值、预测区间、异常分数 |
| 状态 | 未处理、已确认、已忽略 |
| 备注 | 用户本地备注 |

点击一条预警，侧边抽屉展示前后 2 小时曲线、阈值来源、模型和数据版本。

### 6.6 优化决策

#### 输入

- 目标日期；
- 峰平谷时间段和价格；
- 可转移分项；
- 可转移比例；
- 允许移动时间窗；
- 最大延迟；
- 是否基于观测或预测曲线。

#### 输出

- 调整前后负荷曲线；
- 被转移能量的来源和去向；
- 峰值、峰谷差、费用和峰段电量变化；
- 能量守恒校验；
- 假设和不可执行条件。

#### 交互约束

- 转移比例限定在 0% 至 100%。
- 时间窗无可用容量时不强行安排，显示未分配能量。
- 不提供“执行优化”按钮，只提供“运行模拟”。

### 6.7 智能报告

#### 报告生成前

- 显示数据、模型、预测、预警、优化情景的选择范围。
- 允许预览将发送给外部 API 的结构化摘要。
- 显示是否包含原始逐点数据；默认否。
- 要求用户主动点击生成，不在页面加载时自动调用。

#### 报告内容

1. 执行摘要；
2. 数据质量；
3. 用电规律；
4. 预测结果和不确定性；
5. 异常与风险；
6. 优化情景；
7. 可执行建议；
8. 数据与模型限制；
9. 免责声明。

每个关键结论展示证据编号。报告右上角显示“大模型生成”或“本地模板生成”。

#### 操作

- 重新生成；
- 复制；
- 下载 Markdown；
- 查看证据；
- 查看调用诊断；
- 切换为模板报告。

### 6.8 系统设置

分为六个页签：

1. 数据：默认文件、最大上传大小、缺失阈值；
2. 模型：默认模型、设备、批量大小和缓存；
3. 预警：规则阈值和等级；
4. 优化：默认电价与转移约束；
5. 大模型：启用、base URL、模型名、Key 状态、超时；
6. 诊断：版本、CUDA、GPU、目录、数据库和日志。

所有设置显示来源：内置默认、配置文件、环境变量或当前会话。保存后提示哪些设置需要重启。

## 7. 关键组件规范

### 7.1 KPI 卡片

- 标题不超过 10 个汉字；
- 主值、单位、趋势和统计范围齐全；
- 无数据时显示“—”，不显示 0；
- 计算异常时显示错误说明而非隐藏卡片。

### 7.2 图表

- 时间轴按本地数据时间显示，不擅自转换时区；
- 缺失区间断线，不用 0 连线；
- 导出图表前应隐藏交互工具条；
- 同一图最多突出三条主要曲线，其余默认弱化。

### 7.3 表格

- 默认分页或限制高度；
- 支持排序、筛选和下载；
- 数字列右对齐；
- 单位放在列标题；
- 预警证据不可被省略为只有“异常”二字。

### 7.4 操作反馈

- 小于 1 秒：按钮状态变化即可；
- 1 至 10 秒：显示进度或加载提示；
- 超过 10 秒：显示步骤、已用时间和取消入口；
- 成功：说明产物位置或下一步；
- 失败：说明发生在哪一步、数据是否安全、如何恢复。

## 8. 文案规范

推荐：

- “检测到连续 3,723 分钟测量缺失，相关窗口将从训练中排除。”
- “观测值超出 90% 预测区间，此结论表示统计异常，不等同于电气故障。”
- “以下费用为用户输入电价下的情景模拟。”

禁止：

- “系统发现家庭设备已经故障。”
- “模型准确率达到 99%”而没有指标定义。
- “AI 自动优化电网运行。”
- “预计节省 X 元”而没有电价、时段和转移参数。

## 9. 空状态与失败状态

| 场景 | 页面行为 |
| --- | --- |
| 无数据 | 显示加载内置数据和上传入口 |
| 数据校验失败 | 保留问题详情，禁用预测和优化 |
| 无模型 | 允许历史分析，给出训练或加载说明 |
| CUDA 不可用 | 自动切 CPU 并明确提示 |
| 模型不兼容 | 拒绝加载，展示缺少字段或版本差异 |
| API 未配置 | 使用模板报告，提供设置入口 |
| API 超时或限流 | 保留确定性结果并回退，不重复无限请求 |
| 缓存过期 | 标记失效原因并提示重新生成 |
| 导出失败 | 保留页面结果，提示目标目录和权限问题 |

## 10. 展示模式

展示模式的目标是减少答辩现场操作风险：

- 默认加载内置数据和已验证模型；
- 使用预设日期范围；
- 使用缓存预测和缓存报告；
- 放大字号、隐藏开发诊断；
- 关闭耗时训练入口；
- 显示“现场数据回放”而不是伪装成真实传感器流；
- 保留一键恢复和应急重置。
