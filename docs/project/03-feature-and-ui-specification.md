# 03 功能与界面规格

## 1. 设计目标

界面应让非算法专业的教师在 30 秒内理解“当前已经有什么、哪些证据可信、哪些业务尚未实现、下一步可以做什么”。系统采用 Streamlit 多页面单体，吸收 Apple 设计中的即时反馈、空间一致性、克制材质、优质排版和用户控制原则，但不复制 Apple 品牌或官网视觉。重点是信息层级、证据链和演示稳定性，而不是卡片堆叠、装饰性渐变或虚构结果。

## 2. 信息架构

~~~mermaid
flowchart LR
    A["首页总览"] --> B["数据中心"]
    B --> C["用电分析"]
    C --> D["负荷预测"]
    D --> E["监测预警"]
    E --> F["优化决策"]
    F --> G["智能报告"]
    G --> H["系统设置"]
~~~

侧边栏用于全局数据集、时间范围、模型和展示模式；页面内部只放与当前任务相关的筛选条件。

## 3. 全局布局

### 3.1 全局 chrome

- sidebar 顶部保持八页固定顺序和当前位置，底部显示稳定应用身份与当前阶段。
- 导航使用 Streamlit 1.59.2 内置 Material Symbols，不以 emoji 作为主要图标系统。
- 选中项使用低饱和强调和左侧位置指示，不使用整块高饱和背景。
- 顶部 Streamlit chrome 使用克制半透明材质；无 blur 时必须有可读实色回退。
- sidebar 折叠后，页面标题和上下文标签仍能说明当前位置。

### 3.2 页面身份区

- 小型上下文标签；
- 页面主标题；
- 一句说明页面当前能解决的问题；
- 必要时显示阶段或只读边界；
- 标题区保持紧凑，不为每页创建巨大 hero。

### 3.3 页面主体

- 当前阶段使用“页面身份 → 核心状态 → 操作或事实 → 证据 → 边界/下一步”的统一空间路径。
- 数据密集内容可以使用表格和指标，不要求每项放进独立卡片。
- 同类操作在页面间位置一致，控件靠近其影响内容。
- 业务实现后，图表仍须包含：

- 标题；
- 时间范围；
- 单位；
- 聚合口径；
- 悬停说明；
- 数据缺失提示；
- 下载入口或可导出数据的说明。

## 4. 视觉规范

### 4.1 主题与材质

- 深色和浅色均使用低饱和中性背景、稳定正文层和少量语义色。
- 半透明只用于 sidebar、顶部 chrome 或确有浮动关系的区域，正文不使用玻璃叠玻璃。
- 大 surface 数量克制，优先通过留白、对齐和排版建立层级。
- 不使用大面积蓝紫渐变、霓虹描边、持续 glow、循环背景动画和装饰性悬浮物。
- 深浅模式使用独立可读 token；颜色不能成为状态的唯一表达。

### 4.2 集中式语义色

| 含义 | 颜色 | 用法 |
| --- | --- | --- |
| 主色 | accent token | 主按钮、选中状态、焦点与未来主预测线 |
| 成功 | success token | 已验证、已完成、可用 |
| 关注 | warning token | 可继续但需保留证据的 attention 状态 |
| 错误 | error token | 阻断、失败和不可恢复状态 |
| 信息 | information token | 计划、说明和非阻断状态 |
| 禁用/未知 | tertiary text token | 无数据、禁用和不可用 |

同一语义在所有页面使用同一颜色。异常还要配合图标、文字和线型，照顾色觉差异。

### 4.3 字体、排版与数字

- 优先使用平台系统字体和 Windows 中文回退，不新增字体依赖。
- 启用 `font-optical-sizing`，大标题使用更紧 tracking/leading，正文保持舒适行高。
- 间距和字号使用 rem、clamp 与集中 token，150% 缩放下保持空间关系。
- 小字号不得低于可读阈值，也不能用低对比文字制造“精致感”。
- KPI 数字保留合理小数：功率 2 位、电量 1 位、百分比 1 位、费用 2 位。
- 千位使用分隔符；日期统一为 YYYY-MM-DD HH:mm。

### 4.4 交互与辅助功能

- 按钮和可点击控件在 pointer-down / active 时立即反馈，默认压缩到 `scale(0.98)`，约 90ms。
- 非手势 UI 不使用 bounce；当前没有拖拽、sheet、carousel 或 swipe，不为展示动效而凭空增加。
- 页面 rerun 不重复播放大范围入场动画，动效只服务 hover、press 和局部状态变化。
- `prefers-reduced-motion: reduce` 使用短交叉淡化或静态变化，不删除必要反馈。
- `prefers-reduced-transparency: reduce` 关闭 blur 并提高背景不透明度。
- `prefers-contrast: more` 使用更实背景和清晰边界。
- 键盘 `:focus-visible` 使用清晰焦点环，按钮命中区域至少约 44px 高。

## 6. 页面规格

M2.5 完成全局页面身份、布局、导航、状态和交互反馈；M3 已把用电分析页升级为真实业务页。预测、预警、优化和报告仍是后续业务目标，不得因为计划态已经精致就标记为 implemented。

### 6.1 首页总览

#### 目标

在一屏内表达系统当前状态和最重要结论。

#### 顶部 KPI

1. 数据完整度；
2. 当前或最近负荷；
3. 所选区间峰值及发生时间；
4. 所选区间总有功电量；
5. 活跃预警数量；
6. 未来 24 小时预测峰值。

每个 KPI 卡片显示单位、与上一可比区间的变化、数据时间和信息提示。

#### 主内容

- 左侧宽图：历史 24 小时与未来 24 小时负荷，包含预测区间。
- 右上：预警等级分布和最新三条预警。
- 右下：分项用电构成。
- 底部：数据质量摘要、模型卡摘要和“一键生成报告”入口。

#### 状态

- 未选数据：显示引导卡。
- 数据有阻断错误：预测卡禁用，突出数据中心入口。
- 无模型：显示“尚无可用模型”，允许展示历史分析。
- 使用缓存：图表右上明确标记缓存生成时间。

M2.5 当前实现只展示真实数据、模型、GPU、LLM 和 SQLite 状态，以及从 M2 到 M3/M4 的路径；没有提前展示 KPI、预测峰值或预警数量。

### 6.2 数据中心

#### 数据源区

- 内置数据卡：名称、大小、记录数、校验状态、加载按钮。
- 上传区：拖放或选择文件，显示大小限制和必要字段。
- 最近数据集：dataset_id、导入时间、来源和状态。

#### 字段与质量区

- 字段表：原列名、标准字段、类型、单位、非空数、缺失率、最小、最大、均值。
- 时间检查：起止时间、理论频率、重复、断点、乱序。
- 缺失区段：开始、结束、长度、是否可插值。
- 质量卡：完整性、有效性、唯一性、时间完整性、长缺失警告和总状态。

#### 预处理对比

- 原始与清洗后的同区间双线图。
- 处理步骤时间线：解析 → 缺失识别 → 短缺失插值 → 聚合 → 特征生成。
- 每步显示输入行数、输出行数、变化数和配置。

#### 操作

- 运行校验；
- 生成预处理数据；
- 下载质量报告；
- 复制数据指纹；
- 重置为内置数据。

M2.5 当前实现保留完整 SHA-256、主动质量校验、主动预处理、记录/字段/时间范围、缺失行与缺失率、缺失区段、最长缺失、问题、质量状态、15 分钟点数、固定 split、manifest 别名和 12 行处理预览。上传、字段映射、质量报告下载和预处理对比图仍未实现。

### 6.3 用电分析

#### 筛选器

- 日期范围；
- 粒度：15 分钟、小时、日；
- 指标；
- 工作日/周末；
- 是否显示缺失阴影。

#### 图表

1. 总负荷趋势；
2. 小时轮廓曲线及分位带；
3. 星期×小时热力图；
4. 日历热力图；
5. 分项用电堆叠面积图；
6. 相关性矩阵；
7. 高、低、典型日对比。

#### 分析摘要

由确定性规则生成，例如“本区间峰值出现在 20:15”“周末平均负荷比工作日高 X%”。这些摘要是大模型输入证据，不依赖大模型。

#### M3 当前实际实现

- 页面身份区显示“M3 · 确定性历史分析”、用电分析标题、“不训练模型”徽标和当前处理身份。
- 数据依赖分为 ready 与 blocked；查询结果分为 ready、attention、empty 和 failed，均使用统一状态面板而不是单条 warning。
- 日期控件默认最近 30 个完整日，起止日期均包含，内部转换为半开时间区间；控件变化后使用缓存的低成本只读查询自动更新。
- KPI 区显示累计有功电量、平均/峰值/最低有效功率、峰谷时间、覆盖率、理论/实际/缺失点数；未知值显示为“—”，不使用虚构 delta。
- 图表包含总有功功率趋势、小时规律、星期规律、工作日/周末日内对比和分项能量；大范围趋势受 `max_chart_points` 限制并保留缺失断点。
- 周期图 hover 显示有效样本、总样本和覆盖率；星期固定为星期一至星期日。
- 分项结构同时提供条形图和数值表。当前 M2 manifest 记录 11 条负未分项分钟原值，因此展示能量但不生成占比。
- 摘要完全由本地规则生成，明确这是历史确定性分析，没有调用 LLM、训练模型或预测未来。

本阶段未实现粒度切换、无功/电压/电流趋势、日历热力图、相关性、典型日和导出；这些内容继续按需求优先级进入后续分析扩展，不在本次页面中放置无效控件。

### 6.4 负荷预测

#### 控制区

- 预测起点；
- 模型选择；
- 设备选择：自动、CUDA、CPU；
- 区间置信水平，MVP 默认 90%；
- 运行预测或加载缓存。

#### 模型卡

- 模型类型和版本；
- run_id；
- 训练、验证、测试月份；
- 上下文与预测长度；
- 数据指纹；
- 测试指标；
- 是否为当前默认模型；
- 已知限制。

#### 主图

- 历史上下文：灰色实线；
- 真实未来：仅回测时显示，深色实线；
- 点预测：蓝色实线；
- 区间：浅蓝填充；
- 区间外观测：红色点；
- 长缺失区域：灰色阴影。

#### 评估区

- 模型对比表；
- MAE、RMSE、WAPE、sMAPE、R²；
- 1、6、12、24 小时预测误差；
- 按步长误差曲线；
- 区间覆盖率与平均宽度。

不得只显示训练损失。测试区间必须明确。

#### M4 当前实际实现

- 页面从 `models/registry` 加载与当前 `dataset_id`、`preprocess_id`、数据配置指纹、672→96 窗口和 90% 区间等级完全兼容的模型；任何不兼容、缺失或哈希变化都进入 blocked/failed，不做猜测加载。
- 控制区提供 23 个冻结日级测试起点、6 个模型、自动/CUDA/CPU、固定 90% 区间和是否允许缓存；只有点击“运行预测”才读取权重并推理。
- 即时结果与缓存结果使用不同状态标题；缓存 request hash 包含模型、数据、配置、起点、区间与请求设备，损坏缓存被拒绝。
- 模型卡显示 run_id、数据身份、配置指纹、代码提交、固定月份切分、测试 MAE/RMSE、训练设备、耗时、显存和限制。
- 主图真实展示 672 点历史上下文、96 点回测真实值、点预测、90% 共形区间和区间外观测；未知值不会被填 0。
- 评估区显示六模型同起点表、五项测试指标、1/6/12/24 小时累计误差、96 步 MAE/RMSE、区间覆盖率和平均宽度。
- CSV 导出包含预测值、上下界、真实值、区间外标记、forecast/model/run/dataset/preprocess ID、配置指纹、区间等级、时间、状态和实际设备。
- Streamlit 不包含训练循环；正式训练、验证选择、测试评估和共形校准只由命令行脚本完成。

### 6.5 监测预警

#### 回放控制

- 起始日期；
- 速度：1×、10×、60×、逐小时；
- 播放、暂停、单步、重置；
- 当前模拟时间。

#### 实时区域

- 当前负荷、电压、电流；
- 预测中位数与区间；
- 标准化残差；
- 最近 6 小时曲线；
- 当前等级和触发规则。

#### 预警列表

| 字段 | 展示 |
| --- | --- |
| 时间 | 开始与结束 |
| 等级 | 信息、关注、严重 |
| 类型 | 数据质量、规则、预测残差 |
| 证据 | 观测、阈值、预测区间、异常分数 |
| 状态 | 未处理、已确认、已忽略 |
| 备注 | 用户本地备注 |

点击一条预警，侧边抽屉展示前后 2 小时曲线、阈值来源、模型和数据版本。

### 6.6 优化决策

#### 输入

- 目标日期；
- 峰平谷时间段和价格；
- 可转移分项；
- 可转移比例；
- 允许移动时间窗；
- 最大延迟；
- 是否基于观测或预测曲线。

#### 输出

- 调整前后负荷曲线；
- 被转移能量的来源和去向；
- 峰值、峰谷差、费用和峰段电量变化；
- 能量守恒校验；
- 假设和不可执行条件。

#### 交互约束

- 转移比例限定在 0% 至 100%。
- 时间窗无可用容量时不强行安排，显示未分配能量。
- 不提供“执行优化”按钮，只提供“运行模拟”。

### 6.7 智能报告

#### 报告生成前

- 显示数据、模型、预测、预警、优化情景的选择范围。
- 允许预览将发送给外部 API 的结构化摘要。
- 显示是否包含原始逐点数据；默认否。
- 要求用户主动点击生成，不在页面加载时自动调用。

#### 报告内容

1. 执行摘要；
2. 数据质量；
3. 用电规律；
4. 预测结果和不确定性；
5. 异常与风险；
6. 优化情景；
7. 可执行建议；
8. 数据与模型限制；
9. 免责声明。

每个关键结论展示证据编号。报告右上角显示“大模型生成”或“本地模板生成”。

#### 操作

- 重新生成；
- 复制；
- 下载 Markdown；
- 查看证据；
- 查看调用诊断；
- 切换为模板报告。

### 6.8 系统设置

分为六个页签：

1. 数据：默认文件、最大上传大小、缺失阈值；
2. 模型：默认模型、设备、批量大小和缓存；
3. 预警：规则阈值和等级；
4. 优化：默认电价与转移约束；
5. 大模型：启用、base URL、模型名、Key 状态、超时；
6. 诊断：版本、CUDA、GPU、目录、数据库和日志。

所有设置显示来源：内置默认、配置文件、环境变量或当前会话。保存后提示哪些设置需要重启。

## 7. 关键组件规范

### 7.1 KPI 卡片

- 标题不超过 10 个汉字；
- 主值、单位、趋势和统计范围齐全；
- 无数据时显示“—”，不显示 0；
- 计算异常时显示错误说明而非隐藏卡片。

### 7.2 图表

- 时间轴按本地数据时间显示，不擅自转换时区；
- 缺失区间断线，不用 0 连线；
- 导出图表前应隐藏交互工具条；
- 同一图最多突出三条主要曲线，其余默认弱化。

### 7.3 表格

- 默认分页或限制高度；
- 支持排序、筛选和下载；
- 数字列右对齐；
- 单位放在列标题；
- 预警证据不可被省略为只有“异常”二字。

### 7.4 操作反馈

- 小于 1 秒：按钮状态变化即可；
- 1 至 10 秒：显示进度或加载提示；
- 超过 10 秒：显示步骤、已用时间和取消入口；
- 成功：说明产物位置或下一步；
- 失败：说明发生在哪一步、数据是否安全、如何恢复。

### 7.5 统一状态组件

状态语言覆盖 empty、ready、loading、success、attention、blocked、failed、disabled、planned 和 unavailable。每个状态至少包含文字标签、标题、解释；需要时增加证据和下一步。状态不能只依赖颜色，也不能用虚构数据填补空白。

## 8. 文案规范

推荐：

- “检测到连续 3,723 分钟测量缺失，相关窗口将从训练中排除。”
- “观测值超出 90% 预测区间，此结论表示统计异常，不等同于电气故障。”
- “以下费用为用户输入电价下的情景模拟。”

禁止：

- “系统发现家庭设备已经故障。”
- “模型准确率达到 99%”而没有指标定义。

## 9. 空状态与失败状态

| 场景 | 页面行为 |
| --- | --- |
| 无数据 | 显示加载内置数据和上传入口 |
| 数据校验失败 | 保留问题详情，禁用预测和优化 |
| 无模型 | 允许历史分析，给出训练或加载说明 |
| CUDA 不可用 | 自动切 CPU 并明确提示 |
| 模型不兼容 | 拒绝加载，展示缺少字段或版本差异 |
| API 未配置 | 使用模板报告，提供设置入口 |
| API 超时或限流 | 保留确定性结果并回退，不重复无限请求 |
| 缓存过期 | 标记失效原因并提示重新生成 |
| 导出失败 | 保留页面结果，提示目标目录和权限问题 |

## 10. 展示模式

展示模式的目标是减少答辩现场操作风险：

- 默认加载内置数据和已验证模型；
- 使用预设日期范围；
- 使用缓存预测和缓存报告；
- 放大字号、隐藏开发诊断；
- 关闭耗时训练入口；
- 显示“现场数据回放”而不是伪装成真实传感器流；
- 保留一键恢复和应急重置。
