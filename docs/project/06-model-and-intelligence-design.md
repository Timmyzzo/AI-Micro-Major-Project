# 06 模型、预警与智能建议

## 预测

- 输入窗口：672 个 15 分钟点；
- 输出窗口：96 点，即未来 24 小时；
- 基线：Last Value、前一日同刻、前一周同刻、Ridge；
- 对照模型：小型 LSTM；
- 主实验模型：小型单变量 PatchTST；
- 缩放器只在训练集拟合；
- 神经网络早停、Ridge alpha 等内部选参只看验证集；
- 所有候选冻结后在测试集上统一比较和展示；
- 90% 区间由验证残差按预测步长校准。

当前注册表把测试 MAE 最低的前一日同刻标记为页面“展示默认”，其测试 MAE 为 0.6463 kW；PatchTST 测试 MAE 为 0.6622 kW。该展示默认便于页面操作，但不得写成“测试集完全未参与模型选择”的独立结果。若需要严格无偏的部署模型选择，应在查看测试结果前按验证指标确定默认模型，或保留另一段最终留出数据。

项目保留复杂模型未胜出的事实。PatchTST 相对 Last Value 的 MAE 改善为 9.62%，但相对测试 MAE 最低的前一日同刻并未改善，因此只能表述为“超过 Last Value”，不能表述为“超过最佳朴素基线”。

## 回归评价口径

负荷预测输出连续 kW 数值，属于回归任务，不使用分类任务 Accuracy。当前统一展示：

- MAE、RMSE：原始 kW 单位误差，越低越好；
- WAPE、sMAPE：相对误差，越低越好；
- R²：越接近 1 越好，负值表示泛化表现弱于相应均值参照；
- 90% 区间覆盖率和平均宽度：衡量预测不确定性。

若课程需要百分比形式的“准确率”，应定义例如 `|y_pred - y_true| <= 0.5 kW` 的容差命中率，并在报告中同时写出阈值、公式和测试样本范围。

## 预警

- 数据质量预警来自 manifest；
- 规则预警阈值只从训练段 median、MAD 和分位数推导；
- 残差预警由真实值是否落在共形区间外决定；
- 大模型不能改变预警等级。

## 智能建议

外部大模型 API 是正式展示能力，使用一次 Chat Completions 文本生成：

1. 程序先计算聚合摘要；
2. 用户点击按钮；
3. API 最多输出三条中文建议；
4. API 不得补造数值；
5. 成功显示模型回复，失败或空响应显示错误状态。

不实现完整报告、结构化输出、Responses、自动探测、多轮对话、优化建议或报告历史。
