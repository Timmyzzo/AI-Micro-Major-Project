## 训练过程与评价口径

- LSTM 与 PatchTST 均记录 6 个 epoch 的训练损失和验证 MAE；
- 两者最低验证 MAE 都出现在第 1 轮，之后训练损失下降而验证误差上升，训练程序恢复最佳验证权重；
- 当前 Streamlit 负荷预测页提供折叠的训练过程表，可查看逐轮训练损失和验证 MAE；课程报告保留完整训练历史；
- 负荷预测是回归任务，验收指标为 MAE、RMSE、WAPE、sMAPE、R²和区间覆盖率，不使用未定义的分类 Accuracy；
- 当前注册表的“默认模型”是按测试 MAE 标记的页面展示默认，因此不得继续声称测试集完全未参与模型选择；严格无偏的后续版本应在查看测试结果前按验证指标确定部署模型。

## API 验收

自动测试不得访问网络，使用 mock 验证：

- 只调用一次 `chat.completions.create`；
- 请求包含聚合 JSON；
- 不包含原始 CSV 或完整时序；
- 首页连接测试可以显示成功、未配置和失败状态；
- 未配置 Key 时系统仍能启动，首页和智能建议页仍显示操作按钮；
- 空响应和异常显示失败状态；
- 页面加载不发起请求。

最终验收已通过项目运行路径完成一次真实连接与建议生成，返回 `mode=api`、`diagnostic=None`。Key 不记录到仓库或验收文档。
