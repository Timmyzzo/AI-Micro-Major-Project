# 01 项目范围

## 目标

以 UCI 家庭用电公开数据为可复现案例，完成数据处理、历史分析、离线模型训练、24 小时负荷预测、训练过程记录、测试结果与预测区间可视化，并提供历史回放、预警和外部大模型智能建议。课程方法允许在开发阶段选用其他公开电力数据，但更换数据必须完成字段与时间粒度适配，并重新预处理、训练和评估。

## 必须保留

- 当前案例 CSV 的身份和质量校验；
- 缺失段识别、短缺失插值和长缺失保护；
- 固定月份 train/validation/test 切分；
- 历史 KPI、趋势、周期和分项分析；
- 朴素基线、Ridge、LSTM、PatchTST 公平比较；
- LSTM/PatchTST 的 epoch、训练损失、验证 MAE、训练耗时和早停证据；
- 90% 共形预测区间；
- 冻结模型推理、缓存和 CSV 导出；
- 历史回放、质量/规则/残差预警；
- 首页显示大模型状态、模型型号和一键连接测试；
- 用户点击后的一次 Chat Completions 智能建议调用；
- API 配置不完整时不阻断系统其他页面启动。

## 明确删除

- 上传任意 CSV、字段映射和多数据集；
- 优化调度、峰谷电价、负荷转移和真实控制；
- 完整大模型报告系统、HTML 和报告数据库；
- REST、Docker、云端、多用户、通知和真实设备。

## 成功标准

1. 当前案例数据、分析、训练和测试结果可在本机复现。
2. 训练集拟合权重与缩放器，验证集用于早停和内部选参，冻结候选模型在测试集上统一展示最终结果。
3. 报告展示真实训练过程，并使用 MAE、RMSE、WAPE、sMAPE、R²和区间覆盖率解释回归效果，不虚构分类 Accuracy。
4. 模型负结果和评估边界不隐藏；当前按测试 MAE 标记的展示默认不得误写为验证集独立选模结果。
5. API 建议只发送聚合摘要，成功显示模型回复，失败显示明确错误状态。
6. 七页应用可启动，关键页面有自动测试。
7. 最终 Word/PDF 使用真实截图、训练历史和实测指标。
