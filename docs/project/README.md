## 3. 文档权威顺序

发生冲突时按以下顺序处理：

1. 学校课程要求和报告模板；
2. 用户确认的项目范围；
3. 本目录中的需求规格和数据契约；
4. 架构、模型、界面和开发规范；
5. 根目录 README 的摘要性描述；
6. 原基础实验中的示例实现。

原基础实验允许随机抽取 10,000 行后使用 Prophet，但正式项目属于时间序列预测，必须以本项目的数据切分和模型文档为准，不得随机打乱时间顺序。

## 4. 事实、计划和结果的标记

文档统一使用以下语义：

- **已核验**：本轮已从本地文件或课程模板中直接确认。
- **已决定**：项目设计已经锁定，后续实现应遵守。
- **计划**：实现阶段要完成，但当前还没有代码或结果。
- **建议默认值**：可配置的初始值，需通过验证集或实际设备调整。
- **待实测**：不得在报告中写成真实结果。
- **扩展项**：不属于 MVP 验收，时间充足时再做。
